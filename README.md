# Build Your First Agentic Workflow

_Create a GitHub agentic workflow that analyzes repository activity using natural language instructions and safe execution boundaries._

## Welcome

- **Who is this for**: Developers who already know basic GitHub Actions and want to build AI-powered automation.
- **What you'll learn**: How to define an agentic workflow in Markdown, scope permissions and tools, and refine output quality safely.
- **What you'll build**: A repository reporting workflow that generates commit and pull request insights.
- **Prerequisites**:
  - Complete the [Hello GitHub Actions](https://github.com/skills/hello-github-actions) exercise
  - Basic familiarity with repositories, commits, and pull requests

- **How long**: This exercise takes less than 45 minutes to complete.

In this exercise, you will:

1. Create an agentic workflow using Markdown frontmatter.
1. Set execution boundaries with minimal permissions and tool access.
1. Add natural language task instructions, run the workflow, and iterate on results.
